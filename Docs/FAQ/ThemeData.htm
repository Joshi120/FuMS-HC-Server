*******Options*********

·         "Test",  // Theme Name

o   Theme Folder Name.

o   Should contain no spaces or special characters

·         4,        //Mission Selection: 1=Random, 2=In order, 3=Random:once only until all missions run

                           // 4=Static: All missions in 'Mission List' will be created at server start!

                                                 // 5=Static: with no respawn. Use option 5 for 'create once' type missions.

                                                 // Using option 4 will permit mission to run and re-spawn based upon the respawn delay below.

                                                 //  option 5 will let the mission only spawn once per 6 hours (ie server reset)

o   1   = Missions are selected randomly from the *****Mission List**** section.

o   2  = Missions are ran in the order they are found in *****Mission List**** section.

o   3 = Missions are selected randomly and ran. No mission will be repeated until all have been run at least once.

o   4 =  All missions in ****Mission List**** are started at the same time.  Each mission launched runs on its own 'control loop' with the associated 'Respawn delay' from this file.  Thus if there are 6 missions in ****Mission List****, all 6 will be launched.  As each  mission is completed, it will wait the specified Respawn delay, then restart.

o   5 = Similar to option 4, in that all missions in ***Mission List**** will be launched, but the respawn delay is set to 6 hours

·         60, // Respawn delay in seconds

o   Duration FuMS waits before selecting and launching another mission from this theme.

o   This timer is IN ADDITION TO the timer located in each individual mission under its Win and Lose status.

o   If the mission that was launched uses a fixed location there is the potential that the mission will be re-spawned on top of players after they complete the current mission.  If the mission is using a random location (not fixed, not from ****Locations*****) FuMS will attempt to avoid spawning a mission such that players are within its encounter radius at mission start.

·         true, // use Global Loot Data (GlobalLootData.sqf)

o   false = missions run from this theme will use the LootData.sqf located in the theme's folder.

o   This permits Admins to further customize loot to a particular theme.

o   This option permits create of loot packages that are unique to the theme, and not found via 'randomization' in other themes.

·         true,  // use Global Soldier Data file (GlobalSoldierData.sqf)

o   false = missions run from this theme will use the SoldierData.sqf located in the theme's folder.

o   This permits Admins to use specifically customized AI that will only be found in other themes.

·         true // ThemeAutoStart: Setting this to 'false' will prevent normal start-up of the Theme! See \Docs\AdminControls.txt

o   false = Theme will initialize, but will NOT start any missions.

o   When set to 'false' the only method to start this theme is through the FuMS Admin Controls.

o   If FuMS Admin Controls are also disabled, this theme will not run.

 

***** Mission List *****

    // List of Missions.

This is the list of Missions the theme will manage.  Each item in this list should match an associated .sqf in the theme folder.

Example if ["BikeGang"]   is in the list, there should be a BikeGang.sqf in this theme's folder.

There are 2 options that can be included with each mission.  These options are as follows:

o   ["TestMission01",[10700,12200] ]

o   In this example 'TestMission01' will always spawn at the map coordinates [10700,12200]

o   When spawning this mission, no player, plot pole, or other encounter proximity checks are made.

o   ["LandPatrol","Zaros"]

o   In this example 'LandPatrol' will always spawn in the town of 'Zaros'

o   When spawning this mission, no player, plot pole, or other encounter proximity checks are made.

o   Note: Both above options are MAP specific.  So if you take advantage of these options within your theme, it may require the theme to be reconfigured if you move to a different MAP!

o   ["BikeGang"]

o   In this example 'BikeGang' will be spawned at one of the locations in the *****Locations***** section.

o   If *****Locations**** is empty, a random location will be generated. See ****Locations***** for further details

 

***** Locations *****

If no [x,y] or "Name" fixed position is specified for the mission in the *****Mission List ***** section, then one of the locations from this list will be randomly selected.

This list is an array of options. Each element needs to match on of the following formats:

o   ["Villages"] = adds all 'villages' as identified by the map file to the list of locations.

o   ["Cities"] = adds all 'cities' as identified by the map file to the list of locations.

o   ["Capitals"] = adds all 'Capitals' as identified by the map file to the list of locations.

o   ["Marine"] = adds all 'Marine' areas (coastal and bays, water regions) as identified by the map file to the list of locations.

Note: Each of the above are dependent upon what detail the developer of the map put into the map's data files.

Use of the above 'keywords' is universal across all maps. Use of these will require no modifications if the map is changed.

o   ["Zaros"] = adds the specific town/city/capital/marine area to the list of locations. This option is MAP specific.

o   [[10715,10100],"AREA51"] = adds this specific location and its 'name' to the list

o   This option is MAP specific

o   This option permits creation of a static area with a custom name that will be shown on the map (if map option is enabled in the mission)

o   Use of this option with several missions in the theme permits creation of a known geographic area that will possibly contain a random mission, based upon how you configure the rest of the theme.

o   Note any locations here will OVERRIDE use of 'terrain type' checking provided by the "WATER" or "LAND" paramater in the Missions' area configuration section.

o   If no locations are provided in this section for the theme, FuMS will attempt to find a good random location based upon the following paramaters.

o   Not within 'encounter radius' of a plot pole with players present

o   Not within 'encounter radius' of players

o   If "WATER" is specified in the mission file, FuMS will attempt to ensure the entire encounter radius is water.

o   If "LAND" is specified in the mission file, FuMS will attempt to ensure the entire encounter radius is land.

o   15 attempts are made to find a good location. If one is not found, then the last point generated will be used.
